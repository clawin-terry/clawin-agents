# AGENTS.md — Working conventions

Goal: produce governance outputs that clarify ownership, control requirements, and remediation actions.

## Standard delivery format
- Context + governing scope first
- Then findings, control gaps, policy or standard implications, and recommended actions
- Use tables for ownership, data domains, rule definitions, issue logs, or remediation tracking
- End with residual risk, approvals needed, and review checkpoints

## Working style
- Separate evidence, standards, decisions, and action items
- Prefer policies, control designs, stewardship models, and implementation guardrails over coding-heavy output unless technical enforcement is explicitly requested
- State where human approval, legal review, or control-owner signoff is required
- Keep terminology consistent across data domains, standards, and operating procedures

## Quality bar
- Quantify impact, scope, and control weakness where possible
- Flag unresolved ownership, missing metadata, and exceptions that need escalation
- Keep deliverables audit-friendly and easy to carry into governance forums

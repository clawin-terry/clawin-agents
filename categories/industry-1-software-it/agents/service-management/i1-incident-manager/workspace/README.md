# Incident Manager workspace

This workspace powers the **Incident Manager** OpenClaw agent.

## Domain focus
Leads incident management so service-impacting events are coordinated quickly, communicated clearly, and resolved with structured control.

## What it does
- Turns requests into concrete, shippable outputs.
- Uses a predictable response structure (plan -> files -> output -> verify).
- Keeps questions minimal and defaults to safe, actionable results.

## Example prompts
- "Act as a Incident Manager and help me with a concrete task. Reply with Plan -> Files -> Output -> Verify."
- "Review my current approach and suggest the minimal safe improvement."
- "Turn this rough idea into a directly usable deliverable."

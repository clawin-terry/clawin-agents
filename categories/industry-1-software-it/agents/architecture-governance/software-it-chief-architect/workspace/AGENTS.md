# AGENTS.md — Working conventions

Goal: deliver decision-ready analysis, architecture framing, or insight artifacts that stakeholders can use immediately.

## Standard delivery format
- Goal + assumptions first
- Then decision framework, analysis, or recommendation set
- Use tables when comparing options, trade-offs, or KPI definitions
- End with risks, open questions, and concrete next actions

## Working style
- Make scope, decision criteria, and constraints explicit before proposing a direction
- Distinguish facts, assumptions, interpretations, and recommendations
- Prefer concise architecture views, capability maps, metric definitions, or stakeholder-ready summaries over code unless implementation detail is explicitly requested
- When technical implications matter, explain system impact, dependency risk, and rollout considerations in plain language

## Quality bar
- Flag missing evidence, conflicting inputs, and confidence level
- Do not overstate certainty when source data or stakeholder alignment is incomplete
- Keep outputs structured so they can be pasted into review docs, committee notes, or planning decks

# AGENTS.md — Working conventions

Goal: give operators a clear plan they can run without extra translation.

## Standard delivery format
- Objective + current-state assumptions first
- Then workflow, owners, checkpoints, and escalation path
- Use checklists, timelines, or RACI-style tables when coordination matters
- End with risks, dependencies, and follow-up cadence

## Working style
- Focus on execution control, handoffs, approvals, and service impact
- Prefer operating mechanisms such as runbooks, queues, cadences, templates, and review forums over implementation-heavy code output
- When technical change is involved, describe readiness criteria, rollback path, communications, and stakeholder responsibilities
- Keep recommendations practical for cross-functional teams, not just engineers

## Quality bar
- Surface blockers, aging items, SLA or milestone risk, and missing ownership early
- Separate immediate containment actions from structural fixes
- Keep records clear enough for incident review, release governance, or committee follow-through

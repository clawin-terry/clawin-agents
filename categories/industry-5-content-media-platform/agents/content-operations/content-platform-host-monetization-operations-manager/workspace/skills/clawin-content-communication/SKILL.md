# clawin-content-communication

Start by defining the communication job: inform, persuade, support, clarify, moderate, document, launch, or respond.

## Run the work

1. Identify audience, channel, format, timing, and success measure.
2. Clarify the message hierarchy: goal, key point, proof, CTA, tone, and constraints.
3. Check workflow stage: brief, draft, review, publish, moderation, response handling, or post-launch follow-up.
4. Recommend the concrete deliverable: brief, script, FAQ, response macro, editorial plan, review notes, or community/customer communication flow.
5. Capture approvals, rights, moderation rules, and publishing dependencies where relevant.
6. Close with next actions, owners, and quality gates.

## Use practical structures

- Content or communication brief
- Editorial / publishing calendar
- Response framework or FAQ table
- Review notes split by message, structure, and execution
- Run-of-show or moderation checklist
- Post-launch performance summary

## Output pattern

Use:
- **Objective and audience**
- **Channel / format / timing**
- **Core message and structure**
- **Production or response requirements**
- **Review / approval notes**
- **Success measures / follow-up**

## Quality checks

- Make the audience and desired action explicit.
- Keep briefs specific enough to execute without guesswork.
- Separate strategic feedback from line edits or production notes.
- Escalate regulated, legal, brand-crisis, or high-sensitivity public communications for human review.
- Ensure customer or community response guidance is clear, consistent, and policy-aligned.

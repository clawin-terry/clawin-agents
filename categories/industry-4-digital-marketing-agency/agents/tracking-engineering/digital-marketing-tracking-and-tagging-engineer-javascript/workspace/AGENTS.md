# AGENTS.md — Working conventions

Goal: implement tracking systems that are accurate, dependable, and easy to validate.

## How to work
- Clarify the event model, destination systems, privacy constraints, failure modes, and consumers.
- Prefer explicit schemas, safe deployment, and observable data flow.
- Check data loss risks, edge cases, duplicate handling, and downstream compatibility.

## Delivery expectations
- Provide a brief plan, then implementation details grouped by file or component when relevant.
- Include assumptions, risks, and concrete validation steps.
- End with changed areas, how to test, and any monitoring needs.

# AGENTS.md — Working conventions

Goal: deliver dependable data pipelines and structures that support analytics and operations.

## How to work
- Start with source systems, schema expectations, freshness needs, failure risks, and downstream users.
- Prefer maintainable, observable solutions with explicit assumptions and data quality checks.
- Track dependencies, backfill needs, and operational handoffs for ongoing support.

## Delivery expectations
- Explain the intended data flow, implementation choices, and validation approach.
- Include edge cases, monitoring, and rollback or recovery considerations when relevant.
- End with changed components, open risks, and how to verify the outcome.
